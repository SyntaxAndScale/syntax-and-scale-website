# Product Requirements Document: Syntax & Scale LLC Website

## 1. Introduction

This document outlines the product requirements for a single-page corporate website for Syntax & Scale LLC. The website will serve as an online presence to describe the company's services and provide a point of contact.

## 2. Goals and Objectives

*   **Goal:** Establish a professional online presence for Syntax & Scale LLC.
*   **Objective:** Clearly communicate the company's value proposition and services.
*   **Objective:** Provide a straightforward method for potential clients to make contact.
*   **Objective:** Create a simple, low-maintenance website suitable for static hosting (e.g., GitHub Pages).

## 3. Target Audience

*   Potential clients seeking consulting services in marketing/sales automation, analytics, and business process improvement.
*   Individuals or companies interested in technical education, Rust/AI software development, or digital media production.

## 4. Features

### 4.1. Single-Page Layout

The website will consist of a single HTML page with distinct sections.

*   **Header/Navigation:** A persistent header containing the company name/logo and navigation links that scroll to the relevant sections.
*   **Hero Section:** An introduction with a compelling headline and a brief summary of what Syntax & Scale does.
*   **Services Section:** A detailed description of the services offered, translated into marketing-friendly language.
*   **Contact Section:** Information on how to contact the company via email.
*   **Footer:** Standard footer with copyright information.

### 4.2. Core Content

*   **Company Name:** Syntax & Scale LLC
*   **Contact Email:** iw@syntaxandscale.com
*   **Services Description:** The website will elaborate on the following services:
    *   Consulting for revenue growth through Marketing & Sales enablement (automation, analytics, business processes, technical tools).
    *   Technical education.
    *   Software development (specializing in Rust and AI).
    *   Digital media production.

## 5. Non-Functional Requirements

*   **Technology:** The website will be built using only HTML, CSS, and vanilla JavaScript. No backend framework or CMS will be used.
*   **Performance:** The site must be lightweight and fast-loading.
*   **Hosting:** The site must be compatible with static hosting platforms like GitHub Pages.
*   **Maintainability:** The code should be clean, well-structured, and easy to update.

## 6. Assumptions and Dependencies

*   The domain name `syntaxandscale.com` is registered and managed by the user.
*   The user will be responsible for deploying the website files to their chosen hosting provider.
*   The initial version will not include a blog, portfolio, or other dynamic content.
